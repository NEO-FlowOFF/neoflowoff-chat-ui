# MARKDOWN STYLE
> Fonte canônica de estilo para documentação e regras do projeto.
> Todo arquivo deve seguir este sistema de organização visual.

```text
========================================
    ORGANIZATION · MARKDOWN STYLE
========================================
Scope: DOCUMENTATION & RULES
Standard: NΞØ VISUAL SYSTEM
========================================
```

## ⟠ Objetivo

Garantir que toda documentação e regras do projeto sigam o mesmo
sistema de escrita, composição visual e organização semântica.

────────────────────────────────────────

## ⨷ Regras de Formatação (Obrigatórias)

1. **Banner Inicial**: Todo `.md` deve começar com o banner `====`.
2. **Hard Wrap**: Máximo de **80 caracteres** por linha.
3. **Símbolos Unicode**: Use glifos puros (`⟠`, `⨷`, `⧉`, `⍟`).
4. **Blocos Text**: Use para diagramas e tabelas ASCII.
5. **Assinatura**: Feche sempre com o bloco de assinatura oficial.

────────────────────────────────────────

## ⧉ Estrutura Semântica

Todo documento deve ser modular, escaneável e livre de "gordura"
verbal. Trate a documentação como uma interface operacional.

────────────────────────────────────────
